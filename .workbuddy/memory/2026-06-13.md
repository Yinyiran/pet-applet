## 修正：零食全家桶购物车分类（实际实施）

### 背景
上一轮对话中误将需求理解为"AI答题配餐"，实际应为"零食全家桶"（首页的"立即选购"按钮）。

### 本次实际修改

#### 1. js/cart.js 全面重写购物车逻辑
- `cartItems` 初始数据添加 `type: 'normal'`
- `addToCart(emoji, name, price, type)` — 支持 type 参数，默认 'normal'
- `removeFromCart(name, type)` — 按 name + type 精确删除
- `updateQty(name, type, delta)` — 支持 type；专属定制商品增加时检查总数量≥8则阻止
- `renderCart()` — 按分类分组渲染：
  - 普通商品直接列出
  - `type: 'custom'` 商品显示在"🐾 专属定制配餐"分类下
  - 动态提示：已凑齐8份✅（绿色）或 还需凑X份（橙色）
- `renderCartItem(item)` — 新增辅助函数，生成带 type 参数的 HTML
- `toggleItemCheck(checkbox, type, name)` — 新增辅助函数，处理分类复选框
- `addFamilyBucket()` — 改为8款商品，添加时先移除旧的 `type: 'custom'` 商品，再添加新品（均标记 `type: 'custom'`）
  - 8款：磨牙洁齿棒、三文鱼冻干、芝士训练饼干、原切牛肉粒、鸡肉绕钙奶棒、牛骨头磨牙、冻干鸡肝、蔬菜磨牙圈
- `submitCart()` — 新增结算函数：
  - 校验专属定制商品总数量必须恰好8份
  - 普通商品 + 专属定制同时结算时提示"已分成两单结算"
  - 调用支付 toast

#### 2. index.html 修改
- 购物车提交按钮 `onclick` 从 `showToast('调用支付接口')` 改为 `submitCart()`
- "零食全家桶"展示区从4款图改为8款图（4×2网格）
- 标题改为"超值零食全家桶（8款豪礼）"
- 描述文字更新为8款说明

#### 3. style.css 修改
- 添加 `.cart-section-title` 样式（分类标题）
- 添加 `.cart-section-tip` / `.cart-section-tip.ok` / `.cart-section-tip.warn` 样式（动态提示）
- 追加 `.family-bucket-imgs` 4×2 网格布局（覆盖原2×2）

### 关键逻辑
- 专属定制商品总数量恰好8份才可结算
- 增加数量时实时检查，达到8后阻止继续增加
- 减少/删除不受限制（种类可以少于8款）
- 购物车分类显示，普通商品和专属定制商品分开

### 文件清单
- `js/cart.js` — 核心逻辑（全文更新）
- `index.html` — 提交按钮、零食全家桶展示区
- `style.css` — 分类标题/提示样式、全家桶网格布局

## 生成数据库ER图（多次调试Mermaid语法）

### 背景
用户要求根据项目生成数据库ER图，覆盖所有功能，参考行业最佳实践。

### 问题与修复过程
1. **第一次**：使用 Mermaid `erDiagram` 语法，属性类型写了 `varchar(32)`、`decimal(10,2)` 等带括号的类型定义 → Mermaid 10.9.6 不支持，全部报 Syntax Error
2. **第二次**：改用 `flowchart` 语法，节点内用 `<b>` + `<br/>` 显示字段列表 → flowchart 节点不支持 HTML 标签渲染，显示异常
3. **第三次（最终）**：改用 `classDiagram` 语法，属性格式为 `+属性名 : 类型 描述`，每个代码块自包含（跨模块实体在各图中独立定义简化版）

### 最终输出
- 文件：`er-diagram.html`
- 5个分模块ER图 + 1个关系总览图，共6个图表
- 26张表，分5大模块：
  1. 核心业务（用户/宠物/商品/订单）— 10张表
  2. 分佣与邀请系统 — 5张表
  3. AI配餐与答题系统 — 4张表
  4. 同城服务与轻创课程 — 6张表
  5. 积分与账户系统 — 4张表
- 设计原则：第三范式(3NF)，分佣日志等高频场景保留适度冗余

### 关键技术点
- Mermaid `classDiagram` 属性语法：`+attr : type description`（冒号分隔）
- 每个 `<div class="mermaid">` 代码块独立解析，跨块引用的类必须在当前块内定义
- 关系语法：`类名A "基数" --> "基数" 类名B : 关系说明`
